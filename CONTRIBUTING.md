# Contributing to VerdiMobility

Thank you for considering contributing to the VerdiMobility project! We appreciate your efforts in making this project better. Here are some best practices and guidelines to follow when contributing.

## Best Practices for Commits
- **Commit message format**: Use the following format for your commit messages:
  - `type: subject`  
    *Example:* `feat: add user authentication`
- **Types of commits**: Use specific prefixes for commit types:
  - `feat`: A new feature
  - `fix`: A bug fix
  - `docs`: Documentation only changes
  - `style`: Changes that do not affect the meaning of the code (white-space, formatting, etc)
  - `refactor`: A code change that neither fixes a bug nor adds a feature
  - `test`: Adding missing or correcting existing tests

## Pull Requests (PRs)
- **Title and Description**: When creating a pull request, write a clear title and a detailed description explaining what is being done, why, and any relevant context.
  - *Example Title:* `feat: add user login functionality`
  - *Example Description:*  
  "This PR implements user login functionality which allows users to access their accounts securely. It includes the authentication logic and error handling."
- **Link Issues**: Reference any related issues in the PR description by using `#issue-number`.
- **Review Comments**: Be responsive to code reviews and discussions. Address feedback clearly and promptly.

## Issues
- **Creating Issues**: When creating an issue, provide a clear title and a detailed description of the problem. Include:
  - Steps to reproduce the issue
  - Expected behavior
  - Actual behavior
  - Any screenshots or logs that may help
- **Use Labels**: Apply relevant labels to your issues, such as `bug`, `enhancement`, `question`, etc.
